## 📖 剧情设计概述

Life Restart 采用**程序化叙事 + 事件驱动**的剧情设计模式，通过随机事件组合生成独特的人生故事。

### 核心设计理念

1. **程序化叙事**: 通过算法随机组合事件，每次重开都是新的人生
2. **年龄阶段系统**: 不同年龄段触发不同类型的事件
3. **属性影响剧情**: CHR/INT/STR/MNY 属性决定事件走向
4. **多分支结局**: 根据选择和随机因素，导向不同结局

## 🎲 事件触发机制

### 1. 年龄触发

```javascript
// 示例：0 岁事件
{
    "0": [
        {"event": "你出生了，是个男孩。"},
        {"event": "你出生了，是个女孩。"},
        // ...
    ]
}
```

### 2. 属性触发

```javascript
// 示例：高魅力事件
"condition": "CHR>80"

// 示例：高智力事件
"condition": "INT>60"
```

### 3. 随机触发

```javascript
// 概率触发
"probability": 0.3  // 30% 概率
```

### 4. 前置条件触发

```javascript
// 需要特定事件已发生
"condition": "EVT?[事件 ID]"

// 组合条件
"condition": "(CHR>50)&(AGE>18)"
```

## 📝 剧情设计评价

### 优点

1. ✅ **事件丰富**: 3400+ 事件，重复可玩性高
2. ✅ **分支多样**: 不同选择导向不同结局
3. ✅ **现实共鸣**: 反映社会现实，引发思考
4. ✅ **隐藏要素**: 修仙线、长生线增加探索乐趣
5. ✅ **黑色幽默**: 荒诞事件调节气氛

### 不足

1. ❌ **事件重复**: 部分事件内容相似
2. ❌ **逻辑跳跃**: 部分事件衔接不自然
3. ❌ **深度有限**: 部分剧情线挖掘不够
4. ❌ **平衡问题**: 某些天赋过强/过弱

### 改进建议

1. **增加事件关联性**: 前后事件呼应
2. **深化剧情线**: 增加多章节连续剧情
3. **添加更多结局**: 根据人生选择定制结局
4. **增强代入感**: 增加心理描写和内心独白
5. **优化事件触发**: 更合理的概率和条件

## 🎯 总结

Life Restart 的剧情设计采用**程序化叙事 + 事件池**的模式，通过大量随机事件的组合，生成独特的人生故事。

**核心特色**:
- 现实主义与荒诞并存
- 黑色幽默与社会批判
- 多条隐藏剧情线
- 高重复可玩性

**剧情主题**:
- 人生的无常与选择
- 命运与抗争
- 家庭与责任
- 理想与现实
- 超脱与轮回

整体而言，剧情设计成功营造出了"人生重开"的体验，每次重开都是新的人生，符合游戏核心主题。
